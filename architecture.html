03 Call Processing Sequence Diagram

Real-time event flow from the moment a patient calls the Vapi number through transcript extraction, patient identification, Mem[v] memory retrieval, AI summarization, and final dashboard rendering.

%%{init: {'theme': 'dark', 'themeVariables': { 'actorTextColor': '#e0e0f0', 'actorBkg': '#1a2a3a', 'actorBorder': '#2a3a4a', 'actorLineColor': '#3a4a5a', 'signalColor': '#6b6b80', 'signalTextColor': '#d0d0e0', 'labelBoxBkgColor': '#1a1a2a', 'labelBoxBorderColor': '#2a2a3a', 'labelTextColor': '#d0d0e0', 'loopTextColor': '#8b5cf6', 'activationBorderColor': '#3a4a5a', 'activationBkgColor': '#1a2a3a', 'sequenceNumberColor': '#10b981', 'fontSize': '13px' }}}%%
sequenceDiagram
    actor Patient
    participant Vapi as Vapi Voice AI
    participant WH as /api/webhook
    participant Store as In-Memory Store
    participant Gemini as Gemini 2.5 Flash
    participant Process as /api/process
    participant MemV as Mem[v] AI
    participant Nemotron as Nemotron 70B
    participant Orch as Orchestrator
    participant UI as Dashboard UI

    Patient->>Vapi: Calls +1 (669) 310-5333
    Vapi->>WH: speech-update event
    WH->>Store: Store caller phone

    loop Every transcript chunk
        Vapi->>WH: conversation-update / transcript
        WH->>Store: Append transcript line
        WH->>Gemini: Extract (name, location, urgency)
        Gemini-->>WH: Extracted fields
        WH->>Store: Update extracted info
    end

    Note over UI,Store: Dashboard polls /api/debug every 2s

    Vapi->>WH: end-of-call-report
    WH->>Store: Mark call complete

    WH->>Process: POST /api/process (auto-trigger)

    Process->>Store: Step 1: Get caller phone
    Process->>Gemini: Step 2: Analyze full transcript
    Gemini-->>Process: Patient name + details
    Process->>MemV: Step 3: Lookup by phone/name
    MemV-->>Process: Patient record + history
    Process->>MemV: Step 4: Query semantic memories
    MemV-->>Process: Retrieved memories
    Process->>Gemini: Step 5: Generate summary (with Mem[v] context)
    Gemini-->>Process: Complete summary
    Process->>Store: Step 6: Update dashboard data

    Note over Orch: Multi-Agent Pipeline Triggered

    par Wave 1 — Parallel
        Orch->>Gemini: Symptom Parser
        Orch->>Gemini: Vitals Analyzer
        Orch->>Gemini: Lab Interpreter
        Orch->>Gemini: Med Reviewer
    end

    par Wave 2 — Parallel
        Orch->>Gemini: Differential Diagnosis
        Orch->>Nemotron: Pattern Recognition
        Orch->>Gemini: Risk Stratification
        Orch->>Gemini: Comorbidity Analysis
    end

    rect rgb(30, 20, 40)
        Note over Orch,Gemini: Wave 3 — Sequential Validation
        Orch->>Gemini: Cross-Reference Verifier
        Orch->>Gemini: Clinical Guidelines Check
        Orch->>Gemini: Evidence Quality Scoring
    end

    par Wave 4 — Hybrid
        Orch->>Gemini: Treatment Protocol
        Orch->>Gemini: Specialist Referral
    end

    rect rgb(20, 30, 20)
        Note over Orch,Gemini: Wave 5 — Final Review
        Orch->>Gemini: Peer Review Judge
        Orch->>Gemini: Consensus Engine (Ensemble)
    end

    Orch-->>Store: Final diagnosis: 94.2% confidence
    Store-->>UI: Render complete patient intelligence
